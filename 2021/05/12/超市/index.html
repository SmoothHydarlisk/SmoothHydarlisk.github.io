超市

发表于 2021-05-12 分类于 algorithm

题目描述

超市里有 N件商品，每件商品都有利润 pi 和过期时间 di，每天只能卖一件商品，过期商品不能再卖。

求合理安排每天卖的商品的情况下，可以得到的最大收益是多少。

输入格式

输入包含多组测试用例。

每组测试用例，以输入整数 N 开始，接下来输入 N 对 pi 和 di，分别代表第 i 件商品的利润和过期时间。

在输入中，数据之间可以自由穿插任意个空格或空行，输入至文件结尾时终止输入，保证数据正确。

输出格式

对于每组产品，输出一个该组的最大收益值。

每个结果占一行。

数据范围

0≤N≤10000,
1≤pi,di≤1000
最多有 14 组测试样例

题目分析

这道题给我的第一印象是贪心思想，但刚开始时贪心的策略有问题。正确的贪心策略应当是这样的

  • 因为题目说道每天只能卖出一件商品，那已卖出的商品数量就可以代表现在的天数。
  • 应尽量先选取最早过期的商品，否则在保质期长利润低的商品会被保质期短利润高的商品更新掉，这样是不合理的，所以若从保质期短的商品好开始维护结果集合时，我们可以发现大致有以下几种情况
    • 保质期相同，利润高的更新利润低的
    • 由于后来者保质期长，更新时只会更新掉保质期小于等于自己并且利润最小的数据，不会造成数据丢失。

根据上面几点我们可以发现维护集合需要一个小根堆，这里为了更好的理解堆，代码中用数组去模拟了堆的实现。
